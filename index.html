HTML

  • The head element contains information about the webpage.
  • The head element also contains metadata for the web application.
  • Metadata contains important information about accessibility, search engines and performance.
  • Metadata is not typically seen by the user but it helps the browser use data to best serve the webpage to users.
  • The title element is a metadata tag inside the head element.
  • The title element gives a title to the webpage and is required for SEO aka Search Engine Optimization.
  • SEO improves the ranking of your webpage. This means your webpage will show up more often in search engines like Google or Bing.
  • The meta element defines metadata such as the character set, description, keywords, author and viewport.
  • Character set UTF-8 allows you to use most characters and symbols on a webpage.
  • The style element defines CSS styles associated with a webpage.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The script element embeds client side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • The body element helps provide a positive user experience.
  • Elements nested inside other elements are called Child Elements. The element that holds all the children elements is called the Parent element.
  • It is important to use Semantic HTML elements as it makes it easier to read by the browser and other developers.
  • Semantic HTML also makes sure the webpage is accessible to users who may use screen readers or other assistive technologies.
  • Headings help organize a page by being more prominent titles and subtitles.
  • Order matters in headings, make sure to use h1 through h6. Only use h1 once per page.
  • The p element wraps most text on a webpage.
  • The ul element is for unordered lists.
  • The ol element is for ordered lists.
  • The li element is for list items.
  • The img element gives information about images that are displayed on the webpage.
  • The src attribute defines the location/source of the image file.
  • The alt attribute contains a text string that gives a description of the image.
  • The alt text is important for users who use assistive technologies like screen readers.

CSS

  • CSS adds color and style. It defines colors and aesthetics to our webpage.
  • CSS also helps with organization and page layout.
  • CSS will help define style rules for specific elements of an HTML page and format them using a large range of style options.
  • There are 3 ways to style a webpage with CSS; Inline CSS, Internal CSS & External CSS.
  • Inline CSS is useful for quick and permanent changes but it is less flexible than Internal/External CSS as you must edit individually whenever you make a design change.
  • Internal CSS has CSS rules for the webpage inside of the head section of the HTML file. A single change to the CSS rule will apply to all tagged page elements.
  • Inline CSS and Internal CSS are perfectly good methods under the right circumstances but the most common way to style a webpage is External CSS.
  • External CSS makes design changes easier as it allows you to keep all CSS rules in a separate file.
  • Separation of Concerns is a key design principle based on the idea that each section of code should have its own file/responsibility based on its use.
  • A selector defines the element or attributes to which the rules/declarations will apply.
  • Declarations contain 2 important components: CSS property we want to apply & value of the property.
  • Ex: Color:Blue. Color = CSS property & Blue = the value for that property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • When we want to apply a rule to all elements that are visible on a webpage we can use * symbol as a wildcard selector.
  • Two or more elements that we want to style the same can be combined into a single CSS rule separated by a comma so declarations can affect all the elements we want the same.
  • DRY code aja “Do not Repeat Yourself” code encourages us to reduce the number of lines of code. This will reduce the overall size & complextivity of a code base.
  • We can also apply CSS rules to images.
  • After we have created a CSS file we need to link it to our HTML file by using the link element.
  • We can add the link element inside the head element in the HTML file (where the metadata is stored). This will create the connection to our CSS file.
  • Using a Class Attribute as a selector allows us to add styles to certain elements but not all.
  • Class attributes allow us to freely pick and choose which elements will be assigned the CSS rule.
  • Assigning colors in CSS can be done with semantic references like gray/blue or with a hexadecimal code aka hex #.

Git

  • Git is the most commonly used version control software.
  • Git allows you to keep track of each version of your product. We use the command line to access Git.Good for tracking changes locally on a personal computer.
  • Git commands always start with git and are used to save/track versions of our code.
  • Using Git commands we can create a repo, add code to the repo and view different versions of our code which are stored in a unique commit.
  • SHA or Hash is a unique identifier that provides a reference to that specific version of code so we can retrieve it as needed. A web interface of Git.
  • Only need the first 7 numbers/letters of the SHA/Hash to be able to retrieve it.
  • GitHub is a cloud based hosting service that allows us to keep track of our projects inside repositories aka repos
  • Easy to work collaboratively with others and store code.
  • A branch is a part of the workflow process. It is a workspace where you can make changes that dont affect the live site or the main branch.
  • A markdown (.md) file is the way content files are formatted in GitHub. A way to write for the web using plain text and easily converts to HTML.
  • Committing changes is saving a new version of a file. Add/Edit/Delete = making a commit.
  • Pull Request or PR is a way to discuss and review your work with others and have someone review these changes before being merged.
  • After getting approval on a PR you will merge the changes into the main branch.
  • An issue is a way to report new tasks or content fixes, track progress like a project board. Helps track work & make sure to close it once it is done. Often formatted in Markdown language.
  • When naming a new repo remember to be unique & descriptive, use all lower case letters and use dashes as we can use spaces in our repo names.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: add command, adds modification in the current working branch. -A flag, shows we want to add all changes.
  • git pull origin main: pull command to receive branches on the local environment.
  • git push origin feature : Pushes the changes we have locally to our remote branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • JavaScript is case sensitive so myVariable is not the same as myvariable. If there is a problem in your code, check the case.
  • A string is enclosed in quotation marks ” ”.
  • Numbers dont have quotation marks around them.
  • Boolean or True/False values dont need quotation marks around them either as they are special keywords.
  • Each line of JavaScript code ends with a semicolon ;.
  • An operator is a mathematical symbol that gives a result based on two values or variables. This can be addition, subtraction, multiplication or division.
  • Assigned a value to a variable with = Ex: MyVariable = 21.
  • Strict equality operator (===) Will check to see if two values are equal and will return a Boolean result of True if the values are equal and False if they are not equal.
  • Console.log is a method to output a message to the web console by adding an argument.It was created to be easier to test code.
  • In order to link a .js file you would link it inside the body element but using the link element.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript the code runs from the top to the bottom so order counts!
  • We define topics AFTER we tell the computer to log it in the console.
  • We need to be intentional in how we set up code in a JavaScript file.
  • Control flow can be interrupted with a conditional statement & loops to control if the code is executed or to control the order in which code is executed.
  • Conditional statements are code structures to test if an expression returns a true or not.
  • An if statement is a way of introducing decision making into our code. The code will only execute IF the statement meets a condition or if it is true.
  • A value is Falsy if it is one of these values: -0, On, “ “, null, undefined, NoN or a false Boolean.
  • A value is Truthy if it is any other value than above, including a true Boolean.
  • We can have more than one condition by using an else if statement.
  • The computer will continue through all of the conditions that we list. Once it finds a condition that is truthy it will run the code then break out of the if statement.
  • An array is a single variable that allows multiple values to be stored in it. They are typically related in some way.
  • Arrays are useful for storing multiple items with different data types in a single variable.
  • To locate individual data items inside an array we need to find the location of the data item in the array.
  • The location is identified with a unique number called an index.
  • In JavaScript arrays are zero-indexed & sequential which means the first item in an array has an index of 0
  • If we have 4 items in our array they will be 0,1,2,3.
  • Indices will always be whole numbers.
  • We can use less code with an array by using a for loop.
  • A for loop uses the pattern of the indices to do a task on all items in an array by a single code block being executed again and again.
  • To interrupt the control flow we must use the conditional if statement.
  • In order to use the array we need the index and the arrays length. The length is important because the for loop will need to know how many times to run the code until no items are left.
  • Easy to find the length by using the built in property called “length”.
  • Functions are a way of packaging functionality that you might want to reuse so you don't have to type the same code repeatedly.
  • If you see something that looks like a variable name followed by () it is most likely a function.
  • You must call a function by name in order to execute it.
