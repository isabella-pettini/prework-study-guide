HTML

  • The head element contains information about the webpage.
  • The head element also contains metadata for the web application.
  • Metadata contains important information about accessibility, search engines and performance.
  • Metadata is not typically seen by the user but it helps the browser use data to best serve the webpage to users.
  • The title element is a metadata tag inside the head element.
  • The title element gives a title to the webpage and is required for SEO aka Search Engine Optimization.
  • SEO improves the ranking of your webpage. This means your webpage will show up more often in search engines like Google or Bing.
  • The meta element defines metadata such as the character set, description, keywords, author and viewport.
  • Character set UTF-8 allows you to use most characters and symbols on a webpage.
  • The style element defines CSS styles associated with a webpage.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The script element embeds client side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • The body element helps provide a positive user experience.
  • Elements nested inside other elements are called Child Elements. The element that holds all the children elements is called the Parent element.
  • It is important to use Semantic HTML elements as it makes it easier to read by the browser and other developers.
  • Semantic HTML also makes sure the webpage is accessible to users who may use screen readers or other assistive technologies.
  • Headings help organize a page by being more prominent titles and sub-titles.
  • Order matters in headings, make sure to use h1 through h6. Only use h1 once per page.
  • The p element wraps most text on a webpage.
  • The ul element is for unordered lists.
  • The ol element is for ordered lists.
  • The li element is for list items.
  • The img element gives infomation about images that are displayed on the webpage.
  • The src attribute defines the location/source of the image file.
  • The alt atrribute contains a text string that gives a description of the image.
  • The alt text is important for users who use assistive technologies like screen readers.

CSS

  • CSS adds color and style. It defines colors and aesthetics to our webpage.
  • CSS also helps with organization and page layout.
  • CSS will help define style rules for specific elements of an HTML page and format them using a large range of style options.
  • There are 3 ways to style a webpage with CSS; Inline CSS, Internal CSS & External CSS.
  • Inline CSS is useful for quick and permanent changes but it is less flexible than Internal/External CSS as you must edit individually whenever you make a design change.
  • Internal CSS has CSS rules for the webpage inside of the head section of the HTML file. A single change to the CSS rule will apply to all tagged page elements.
  • Inline CSS and Internal CSS are perfectly good methods under the right circumstances but the most common way to style a webpage is External CSS.
  • External CSS makes design changes easier as it allows you to keep all CSS rules in a separate file.
  • Separation of Concerns is a key design principle based on the idea that each section of code should have its own file/responsibility based on its use.
  • A selector defines the element or attributes to which the rules/declarations will apply.
  • Declarations contain 2 important components: CSS property we want to apply & value of the property.
  • Ex: Color:Blue. Color = CSS property & Blue = the value for that property.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • When we want to apply a rule to all elements that are visible on a webpage we can use * symbol as a wildcard selector.
  • Two or more elements that we want to style the same can be combined into a single CSS rule separated by a comma so declarations can affect all the elements we want the same.
  • DRY code aja “Do not Repeat Yourself” code encourages us to reduce the number of lines of code. This will reduce the overall size & complextivity of a code base.
  • We can also apply CSS rules to images.
  • After we have created a CSS file we need to link it to our HTML file by using the link element.
  • We can add the link element inside the head element in the HTML file (where the metadata is stored). This will create the connection to our CSS file.
  • Using a Class Attribute as a selector allows us to add styles to certain elements but not all.
  • Class attributes allow us to freely pick and choose which elements will be assigned the CSS rule.
  • Assigning colors in CSS can be done with semantic references like gray/blue or with a hexadecimal code aka hex #.

Git

  • git add -A: add command, adds modification in the current working branch. -A flag, shows we want to add all changes.
  • git pull origin main: pull command to receive branches on the local environment.
